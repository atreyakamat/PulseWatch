4Ô∏è‚É£ Core Features Breakdown
4.1 Website Management
Capabilities

Add single URL

Bulk add (comma-separated / newline-separated)

Edit / Pause / Delete monitors

Set check frequency (in minutes)

Website Model
Website {
  id: UUID
  name: string
  url: string
  frequency: number // in minutes
  isActive: boolean
  createdAt: Date
}

4.2 Monitoring Engine
How It Works

Uses node-cron to schedule checks

Each website has its own frequency

Sends HTTP GET request

Tracks:

Status code

Response time

Timeout

Errors

Health Check Logic
1. Send request
2. Measure response time
3. Check status (200‚Äì399 = UP)
4. If DOWN:
   - Log event
   - Trigger alert

4.3 Logging System
Log Model
UptimeLog {
  id: UUID
  websiteId: UUID
  status: "UP" | "DOWN"
  responseTime: number
  statusCode: number
  errorMessage: string | null
  checkedAt: Date
}

Stored Events

Every check (optional sampling)

All downtime events (mandatory)

4.4 Email Alert System
Alert Rules

Trigger email when:

Site goes DOWN

Optional: site comes back UP

Avoid spam using cooldown window

Email Config Model
AlertConfig {
  id: UUID
  email: string
  isEnabled: boolean
}
